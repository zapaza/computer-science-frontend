# Cхема кодирования текста

## 1. Базовый набор символов (5 бит)
Для 29 самых частых символов используются 5-битные коды. Код `11111` выделен под `ESC`.

| Биты  | Символ | Биты  | Символ |
| ----- | ------ | ----- | ------ |
| 00000 | пробел | 10000 | у      |
| 00001 | о      | 10001 | я      |
| 00010 | е      | 10010 | ы      |
| 00011 | а      | 10011 | ь      |
| 00100 | и      | 10100 | г      |
| 00101 | н      | 10101 | з      |
| 00110 | т      | 10110 | б      |
| 00111 | с      | 10111 | ч      |
| 01000 | р      | 11000 | й      |
| 01001 | в      | 11001 | х      |
| 01010 | л      | 11010 | ж      |
| 01011 | к      | 11011 | ш      |
| 01100 | м      | 11100 | . (точка) |
| 01101 | д      | 11101 | , (запятая) |
| 01110 | п      | 11110 | резерв |
| 01111 | резерв | 11111 | ESC    |

*Уточнение: `11111` всегда означает переход к расширенным кодам.*

## 2. Расширение ESC (3 дополнительных бита)

Если встречен код `11111`, следующие 3 бита определяют категорию:

| Код после ESC | Категория | Описание |
| ------------- | --------- | -------- |
| 000 - 110     | Редкие буквы | э, щ, ц, ю, ф, ё, ъ |
| 111           | EXTENDED  | Управляющие команды и блоки |

## 3. Таблица редких букв (ESC + 3 бита)

| Код | Символ |
| --- | ------ |
| 11111 000 | э |
| 11111 001 | щ |
| 11111 010 | ц |
| 11111 011 | ю |
| 11111 100 | ф |
| 11111 101 | ё |
| 11111 110 | ъ |

## 4. Блок EXTENDED (ESC + 111 + тип)
Типы данных внутри расширенного блока:
- `00`: **CAPS_NEXT** (Следующая буква заглавная)
- `01`: **CAPS_LOCK** (Переключение регистра)
- `100`: **DIGIT_ONE** (Одна цифра, +4 бита)
- `101`: **DIGIT_RUN** (Серия цифр, +Elias Gamma + 4 бита на цифру)
- `110`: **PUNCT** (Пунктуация, +4 бита)
- `111`: **CONTROL** (Управление, +2 бита: `00`=\t, `01`=\n, `10`=EOF)

## 5. Работа с регистром (CAPS)

### 5.1. Одиночная заглавная (CAPS_NEXT)
Код: `11111 111 00` + код буквы.
Используется для первой буквы в предложении или имён собственных.

### 5.2. Режим CAPS LOCK
Код: `11111 111 01` + `X`, где `X`:
* `1` — Включить (все последующие буквы интерпретируются как заглавные).
* `0` — Выключить (возврат к строчным).

## 6. Кодирование чисел

Кодировщик заглядывает вперёд: если цифра одна — `DIGIT_ONE`, если две и более — `DIGIT_RUN`.

### 6.1. Одиночная цифра (DIGIT_ONE)
Формат: `ESC + EXTENDED + 100 + 4 бита цифры`.
Пример для `5`: `11111 111 100 0101` (всего 12 бит).

### 6.2. Серия цифр (DIGIT_RUN)
Формат: `ESC + EXTENDED + 101 + длина (Elias Gamma) + цифры (по 4 бита)`.
Используется только для 2 и более цифр.

## 7. Прочая пунктуация (PUNCT)
Формат: `ESC + EXTENDED + 110 + 4 бита кода`.

| Код | Знак | Код | Знак |
| --- | ---- | --- | ---- |
| 0000 | !    | 1000 | )    |
| 0001 | ?    | 1001 | "    |
| 0010 | :    | 1010 | «    |
| 0011 | ;    | 1011 | »    |
| 0100 | -    | 1100 | …    |
| 0101 | (    | 1101 | №    |
| 0110 | /    | 1110 | \    |

## 8. Управляющие символы и EOF
Формат: `ESC + EXTENDED + 111 + 2 бита`.

| Код | Значение |
| --- | -------- |
| 00  | `\t` (табуляция) |
| 01  | `\n` (перевод строки) |
| 10  | **EOF** (конец файла) |
